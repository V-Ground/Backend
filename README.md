## 논의

- 기관 관리자 : 강사만 관리한다.
- 강사 : 학생들만 관리한다.
- 한 줄 요약 : **계층 구조라서 굉장히 조직적이다.**
- 장점
  - 개발이 쉬워진다 (초대링크 하나만 보내면 되므로)
  - 기관이 학생을 등록하는 일이 학생에게 위임된다.
- 플로우
  - 기관 관리자가 강사 등록용 배포 링크 생성
  - 강사는 해당 배포링크를 클릭해서 회원가입 진행
    - 강사를 식별할 수 있는 방법
      - username-bob10
    - 관리자가 강사 회원가입을 검증해야한다.
  - 강사가 학생 추가
    - 강사는 학생 초대 링크를 생성한다.
    - 학생은 초대 링크로 들어가서 회원가입 진행
      - 경우가 나뉨 (버튼 두 개를 보여주고 클릭하도록)
        - 학생이 이미 우리 회원인 경우
          - 로그인하고 바로 수강신청
        - 학생이 우리 회원이 아닌 경우
          - 회원가입 후 수강신청

- 한 줄 요약 : **일단은 모르겠다.**
